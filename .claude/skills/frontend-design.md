# Frontend Design Skill

## Purpose
Before writing any frontend code, commit to a specific visual direction. Answer these four questions first, out loud, before touching any file:

1. **Purpose** — What is this UI trying to make the user feel or do?
2. **Tone** — What visual personality fits? (warm/clinical/playful/editorial/luxury)
3. **Constraints** — What must stay consistent with the existing design system?
4. **Differentiation** — What one design choice will make this feel distinctive, not generic?

## Rules
- State your font choice before writing any component. Never use Inter, Roboto, or Arial.
- State your color palette before writing any component. Never use generic purple-on-white.
- Commit to a specific aesthetic direction. Do not hedge with "clean and minimal."
- Reference buddyguard.bg as the quality bar for warmth and polish.
- Every interactive element must have a visible hover state.
- Every section must have an entrance animation (CSS only, no JS libraries).
- Mobile-first. Check 375px width before considering any component done.

## Trigger
Auto-load this skill whenever working on any component, page, layout, or styling task.
