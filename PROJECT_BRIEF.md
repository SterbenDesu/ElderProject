## MVP goal

Build the smallest deployable version of the app that proves the core user flow works.

## Core user flow

1. User lands on the homepage.
2. User understands what the app does.
3. User signs up or logs in, if authentication is needed.
4. User completes the main action.
5. User sees a useful result.
6. User can return later and continue using the app.

## Non-goals for MVP

These should not be built until the MVP is working:

* Complex admin systems
* Advanced analytics
* Payment flows
* Mobile apps
* Multi-language support
* Complex notification systems
* Over-engineered architecture

## Design requirements

* Simple layout.
* Clear navigation.
* Responsive on mobile and desktop.
* Loading states for async actions.
* Error messages that explain what went wrong.
* Empty states where the user has no data yet.

## Technical requirements

* Follow the existing repository stack.
* Keep dependencies minimal.
* Prefer boring, stable tools.
* Add deployment documentation.
* Add tests only where the repository already has a test setup, unless explicitly asked.
